# Contributing

这个仓库只接受“创业和社交场景里的处事规则、原创场景、评测题、方法文档”和 skill 改进。

## 可以提交

- 原创创业现实场景题
- 原创社交现实场景题
- 地推、BD、求人、谈合作、借资源、饭局、客户局、团队局案例
- 校园代理、渠道扩张、现金流、品牌、产品上市、资本化路径案例
- 同学局、亲戚局、公开夸奖、拒绝、求助、尴尬场面案例
- 买单、请客、账单、AA、被推成冤大头这类饭局案例
- 亲属争执、家事旁听、被要求评理这类撤退边界案例
- 单方面付出、关系投入、客户试点参与、朋友互助这类投入结构案例
- 喜事场、主家场面、低成本补位、挡酒接场这类场面价值案例
- 对某个场景的行为规则抽取
- 更自然的中文话术模板
- 评测题和通过标准
- 对 skill 触发条件、输出格式、风险边界的改进

## 不要提交

- 小说全文
- 章节正文
- 长段原文摘录
- 可替代阅读原书的剧情整理
- 侵犯版权的资料包

## 新增规则格式

新增规则最好包含：

1. 场景
2. 错误方向
3. 更像陈汉生的动作
4. 为什么这样做
5. 风险边界
6. 可复用规则

重点不是把话说漂亮，而是解释清楚谁有权、谁承担风险、这句话抬了谁、避开了谁、自己获得了什么现实利益。

如果提交发家路径类内容，还要额外说明：

1. 当前动作赚到的是什么
2. 当前动作换到了什么下一阶段资产
3. 这一步为什么能让下一张牌桌更容易进

如果提交买单类内容，还要额外说明：

1. 这顿钱能不能买到局
2. 买单能换到什么身份、人脉、资源或下一次见面
3. 如果不能买局，为什么只接协调权、不接全责

如果提交亲属争执类内容，还要额外说明：

1. 自己是不是这个家庭内部的人
2. 留下会不会变成证人、裁判或情绪垃圾桶
3. 撤退时怎么给面子，而不是让对方觉得你在嫌弃家事

如果提交关系投入类内容，还要额外说明：

1. 谁一直在付出
2. 对方有没有主动选择、投入时间、投入资源或承担一点成本
3. 怎么让对方从接受供给变成参与关系

如果提交场面帮忙类内容，还要额外说明：

1. 被帮的人是不是主家、自己人、关键人物或未来入口
2. 帮忙能不能稳住场面、抬住对方、换到人情
3. 成本是否可控，是否会变成接锅、背债或当裁判
